# Murmur Figma 原型 QA Review

本文档用于审查现有 Figma 原型中与「底部栏遮挡、文本异常、卡片重叠、长文案溢出」类似的视觉问题，并给出统一修复规则。

当前 Figma 文件：

[Murmur AI 省力账本 - Mobile PRD Prototype](https://www.figma.com/design/7PGVYEttzthUwNOhVDgFym)

注意：本 Review 中发现的问题已在 Figma 文件中新增 `Legacy Mobile QA Fixed` 源页面完成修复，并已整合进 `00 Complete Prototype Entry - Start Here` 主入口页。

## 1. QA 结论

现有旧版移动端原型中，确认存在 2 个需要修复的布局问题：

1. **严重问题：`06 Stats - 统计趋势` 底部排行卡片覆盖 Tab Bar。**
2. **中等问题：`03 Quick Record - 记录一次 AI 使用` 的「修改/返工」滑块超出时间成本卡片。**

此外，`06 Stats` 还有两个潜在视觉风险：

- 紧凑指标卡中的 `+210m` 文本容器偏窄，可能导致换行或显示拥挤。
- 净收益趋势图中负值柱可能与日期标签重叠。

其他旧版移动端画板未发现同等级的底部栏遮挡问题，但仍建议按本文档的统一安全区规则重新检查。

## 2. 全局 QA 规则

### 2.1 底部 Tab Bar 安全区

旧版移动端画板尺寸为 `390 x 844`，底部 Tab Bar 位置为：

```text
Tab Bar y = 768
Tab Bar height = 76
```

所有非 Tab 内容必须满足：

```text
contentBottom <= 752
```

解释：

- `768` 是 Tab Bar 顶部。
- `752` 给 Tab Bar 上方保留至少 `16px` 安全距离。
- 任何列表、按钮、图表、卡片、文本都不能超过 `y=752`。

如果是可滚动页面：

- 可滚动内容可以超过屏幕高度，但底部必须有 `bottom padding >= 96`。
- 固定 Tab Bar 必须在滚动内容上方保持独立层级。
- 最后一条列表滚动到可见区域时，不能被 Tab Bar 遮挡。

### 2.2 无底部栏页面安全区

无 Tab Bar 的移动端页面仍需保留底部安全距离：

```text
primaryActionBottom <= 804
```

说明：

- 画板高度 844。
- 底部至少保留 40px，用于设备安全区和圆角。

### 2.3 长中文标题规则

中文标题在窄卡片中必须满足：

- 字号 13-14 时，文本框宽度不少于 `文字数 * 14 + 16`。
- 重要标题禁止被底部栏、按钮、卡片边界覆盖。
- 固定 line height，推荐：
  - 13px 字号：line height 17。
  - 14px 字号：line height 18。
  - 18px 字号：line height 24。
- 不能让长中文词组处在刚好与覆盖层相交的位置。

### 2.4 紧凑指标卡规则

宽度小于 120 的指标卡：

- 数值字号不超过 24。
- 数值文本框宽度至少 88。
- 超过 5 个字符的数值使用短单位：
  - `+210m` 可以改为 `+3.5h` 或将卡片宽度增加。
  - `2h 18m` 不适合放在 104 宽卡片中。

### 2.5 图表标签规则

柱状图存在负值时：

- x 轴标签不得与负值柱重叠。
- 正负柱图需要单独给负值区域留高度。
- 标签应放在图表底部固定区域，例如 chartBottom + 12。
- 如果图表容器高度不足，优先减少柱高比例或增加图表容器高度。

### 2.6 组件防重叠规则

每个移动端画板交付前必须做以下检查：

```text
for each visible node:
  nodeBottom = node.y + node.height
  if screen has Tab Bar:
    assert nodeBottom <= 752 OR node is Tab Bar
  else:
    assert primaryActionBottom <= 804
```

对文本节点还要检查：

```text
textRight <= parentRight - 12
textBottom <= parentBottom - 8
lineHeight is explicit
```

## 3.3 03 Quick Record - 记录一次 AI 使用

状态：需要修复。

问题：

- 时间成本卡片位于 `y=322`，高度 `224`，底部 `546`。
- 「修改/返工」文本位于 `y=546`。
- 对应滑块位于 `y=570`。

影响：

- 用户会误以为第四个时间字段不属于时间成本区块。

推荐修复：

```text

时间成本卡片 height = 236

Prompt 时间 y = 388
审核时间 y = 438
修改/返工 y = 488
结果质量标题 y = 574
结果质量 chips y = 604
感受标题 y = 656
感受 chips y = 686
本次净收益 y = 748
保存按钮 y = 736
```

进一步建议：

- Quick Record 应使用 ScrollView。
- 保存按钮可以做底部 sticky action bar，避免不同字体大小下被挤压。

## 3.6 06 Stats - 统计趋势

状态：严重问题，需要优先修复。

### 问题 A：场景排行覆盖底部 Tab Bar

已知布局：

```text
Tab Bar y = 768
场景排行标题 y = 638
第一条卡片 y = 672, height = 58, bottom = 730
第二条卡片 y = 740, height = 58, bottom = 798
```

结论：

第二条卡片底部 `798` 明显超过 Tab Bar 顶部 `768`，发生覆盖。

用户指出的「复杂方案生成」显示异常也来自这里：

- 标题文本位于第二条卡片顶部区域。
- 其下半部分与底部 Tab Bar 的覆盖区域接近或相交。
- 视觉上出现「复杂」两个字异常。

推荐最终修复：

```text
场景排行标题 y = 618
第一条卡片 y = 648
第二条卡片 y = 710
卡片高度 = 52
第二条卡片 bottom = 762
Tab Bar y = 768
安全间距 = 6
```

更稳妥版本：

```text
场景排行标题 y = 612
第一条卡片 y = 640
第二条卡片 y = 700
卡片高度 = 52
第二条卡片 bottom = 752
安全间距 = 16
```

建议采用更稳妥版本。

### 问题 B：紧凑指标卡数值可能拥挤

已知布局：

- 三张顶部指标卡宽度 `104`。
- 内部数值文本宽度约 `76`。
- `+210m` 使用 28px 粗体。

风险：

- `+210m` 在 76px 宽文本框内可能换行或挤压。

修复建议：

- 将紧凑指标卡数值字号从 `28` 改为 `24`。
- 数值文本框宽度从 `76` 增加到 `84`。
- 或将三张指标卡改为两列布局。

### 问题 C：负值柱可能与日期标签重叠

已知布局：

- 图表基线 y = `430`。
- 负值柱从 `y=430` 向下绘制。
- 日期标签位于 `y=446`。

风险：

- 负值柱底部可能覆盖日期标签。

修复建议：

- 将日期标签下移到 `y=454`，同时保证图表容器底部足够。
- 或将负值柱绘制区域压缩，让负值柱底部不超过 `y=442`。
- 推荐做法是图表内部预留标签区：

```text
chartPlotTop = 356
chartAxisY = 424
chartLabelY = 450
chartBottom = 466
```

## 4. 新版自动检测原型的 QA 要求

新版自动检测原型还未实际写入 Figma 文件。创建新版画板时必须遵守以下规则。

### 4.1 macOS 桌面画板

桌面画板尺寸：

```text
1360 x 860
```

规则：

- 主内容区域底部至少保留 `40px`。
- Sidebar 固定宽度不小于 `220px`。
- 表格或列表不得贴到窗口底部。
- 操作按钮不得低于 `y=780`。
- 所有卡片应有至少 `16px` 间距。

### 4.2 Android 移动画板

Android 画板尺寸：

```text
390 x 844
```

如果有底部导航：

```text
Tab Bar y = 768
contentBottom <= 752
```

如果没有底部导航：

```text
primaryActionBottom <= 804
```

列表页面必须：

- 使用 ScrollView。
- bottom padding >= 96。
- 最后一条内容滚动到可见区域时不能被底部栏遮挡。

### 4.3 Browser Extension Popup

Popup 建议尺寸：

```text
360 x 560
```

规则：

- 底部按钮区底部不超过 `520`。
- 每个按钮高度不小于 `40`。
- 当前会话卡片文案最多 3 行。
- 域名和工具名不能重叠。

## 6. 修复后验收方式

修复完成后，对每个移动端画板做以下验收：

1. 打开画板截图。
2. 检查所有非 Tab 内容底部是否小于 `752`。
3. 检查每个中文标题是否完整、无裁切、无覆盖。
4. 检查所有按钮是否至少 40px 高。
5. 检查所有卡片之间至少 8px 间距。
6. 检查列表最后一项是否不会进入底部栏。
7. 检查图表标签是否不与柱子重叠。

如果 Figma 工具可用，建议用以下流程：

```text
get_metadata
→ 自动扫描每个 frame 中的节点位置
→ get_screenshot
→ 人眼复核重点画板
→ use_figma 写入修复
→ get_screenshot 二次确认
```
